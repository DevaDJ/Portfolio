Dirty Frag

A Silent 9-Year Linux Kernel Flaw Enabling Deterministic Root Access

Published: May 2026 • Severity: Critical • Impact: Local Privilege Escalation → Full Root Compromise

Executive Overview

⚠ Critical Security Alert

Dirty Frag is a newly disclosed Linux kernel vulnerability chain that enables reliable local privilege escalation to root without race conditions.

Unlike previous kernel exploits such as Dirty COW and Dirty Pipe, Dirty Frag delivers deterministic exploitation, making attacks highly reliable and easy to automate.

  • Existed undetected for approximately 9 years
  • Public proof-of-concept already available
  • No race condition required
  • Deterministic and scriptable exploitation
  • Affects Linux kernels released between 2017 and May 2026

What Is Dirty Frag?

Dirty Frag is a vulnerability chain composed of two independent Linux kernel flaws affecting networking subsystems.

CVE Component Issue Type
CVE-2026-43284 xfrm-ESP (esp4 / esp6) Page-cache write flaw
CVE-2026-43500 RxRPC Page-cache write flaw

These vulnerabilities allow attackers to corrupt page-cache-backed memory and modify protected executable mappings without altering files on disk.

Why This Vulnerability Matters

Key Characteristics

  • No race condition required
  • Deterministic exploitation
  • Stealthy memory-only modification
  • Wide exposure across Linux distributions

Technical Insight

At a high level, exploitation follows the sequence below:

  1. Attacker gains local code execution.
  2. Targets vulnerable networking modules.
  3. Manipulates fragmented page-cache memory.
  4. Overwrites protected executable mappings.
  5. Escalates privileges to root.

The critical innovation lies in fragment-aware page-cache manipulation, enabling precise and repeatable memory corruption.
